# 敬一书院预约系统运维监控方案

## 一、日志管理
- 服务端日志：PM2输出到 `/var/log/jingyi/`
- Nginx日志：`/var/log/nginx/`
- 日志轮转：logrotate每日压缩，保留30天

## 二、监控告警
- 服务存活：PM2自动重启 + 健康检查接口
- 磁盘空间：>80%告警
- 内存使用：>512M自动重启
- API响应时间：>1s告警
- 错误率：>5%告警

## 三、数据备份
- MySQL：每日全量备份，保留7天
- 上传文件：每周备份到对象存储
- .env配置：版本控制（加密）

## 四、版本迭代流程
1. 开发分支 → 测试环境验证
2. Code Review → 合并到main
3. 部署脚本自动拉取和重启
4. 小程序版本更新需提交审核

## 五、安全维护
- 定期更新npm依赖（每月）
- JWT密钥定期轮换（每季度）
- SSL证书续期（Let's Encrypt自动续期）
- 日志审计（每周检查异常请求）

## 六、小程序审核材料清单
1. 小程序功能说明文档
2. 隐私政策和用户协议
3. 用户信息处理声明
4. 服务器域名备案证明
5. 管理员账号测试信息
6. 功能截图和录屏
