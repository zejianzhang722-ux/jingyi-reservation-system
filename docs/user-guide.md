# 用户使用手册

## 1. 学生端

### 登录与资料

1. 在微信小程序中完成登录；
2. 首次使用时核对姓名、学号、楼栋和联系方式；
3. 进入“我的”页面可查看信用分、限制状态和通知；
4. 登录失效时按页面提示重新登录，不要重复快速点击。

### 浏览功能房

- 首页可按类型、楼栋和关键词查找功能房；
- 房间详情展示位置、容量、设施、开放时间、审批要求和座位；
- 选择日期后查看时间轴，绿色或可选状态表示当前可预约；
- 页面显示的占用状态以提交预约时的服务器最终校验为准。

### 创建预约

1. 选择房间、日期和起止时间；
2. 自习室还需选择座位；
3. 填写用途和人数；
4. 核对预约确认页后提交；
5. 看到成功结果后不要再次提交。

出现“时间段已被占用”说明其他用户先完成了预约，应重新选择时间或座位。网络超时后可返回“我的预约”确认是否已创建，客户端会使用幂等机制避免同一请求重复生成预约。

### 候补

预约已满时，可在允许的房间和时间段加入候补。原预约被取消或拒绝后，系统会按队列尝试转为正式预约，并通过站内通知提示结果。

### 审批

部分房间预约会进入管理员或辅导员审批。审批前状态为“待审批”，通过后才可使用。被拒绝时可在预约详情查看原因。

### 签到与签退

1. 预约临近开始时打开预约详情；
2. 向现场工作人员出示动态签到码；
3. 动态码短时有效，失效后重新获取；
4. 使用结束后按要求签退；
5. 不要转发签到码，已使用或过期的凭证不能再次使用。

### 取消预约

在规定时间前可取消预约。临近开始时间取消、未签到或违规使用可能影响信用分。取消成功后，系统会释放座位并处理候补队列。

### 通知和信用分

- 通知页面显示预约审批、候补、签到和系统消息；
- 信用分页面显示每次加减分原因；
- 对记录有异议时，可通过反馈功能提交预约编号、时间和说明。

## 2. 小程序管理员端

管理员只能处理角色和楼栋范围内的数据。

### 待审批预约

1. 进入管理员首页查看待审批数量；
2. 打开预约详情，核对用户、房间、时间、用途和人数；
3. 通过或填写明确原因后拒绝；
4. 页面提示预约已被处理时，刷新后查看最新状态，不要连续重复提交。

### 现场签到

- 扫描学生出示的动态凭证；
- 核对房间、预约人和时间；
- 扫码异常时可在权限允许范围内人工签到；
- 重复签到会被系统拒绝；
- 巡查发现异常时按实际情况记录。

### 房间和座位管理

- 临时维护应修改房间或座位状态；
- 修改开放时间、容量和审批要求前确认不会影响已存在预约；
- 不得通过删除房间规避历史数据问题。

## 3. 网页管理后台

### 登录

使用管理员账号登录。超级管理员可管理全局账号、楼栋、系统配置、操作日志和备份；普通管理员只能访问授权楼栋。

### 预约管理

- 使用日期、状态、房间和用户条件筛选；
- 审批、拒绝和取消操作会记录操作者与请求编号；
- 导出数据前确认筛选范围；
- 遇到 409 提示说明记录已被其他管理员更新，应刷新页面。

### 数据统计

统计模块包含仪表盘、预约趋势、房间使用率、高峰时段、爽约和用户数据。大范围导出应避开预约开放高峰。

### 公告、反馈和系统配置

- 公告发布前核对标题、正文、置顶和发布状态；
- 反馈处理后填写清楚的回复；
- 修改预约提前天数、取消限制和信用规则前，应评估对现有预约的影响并留存操作说明。

### 备份与运维

超级管理员可创建真实加密备份、查看执行记录和验证完整性。管理后台不会直接提供备份文件下载。恢复操作只能由服务器运维人员按灾备手册执行。

## 4. 常见问题

### 页面一直加载

检查网络后重新进入页面。管理员可查看运维状态，确认数据库、Redis和接口是否就绪。

### 提交预约后没有看到结果

先查看“我的预约”。不要连续使用不同参数重复提交；确需重试时应保持同一预约内容。

### 无权访问某个房间或预约

管理员账号可能只绑定了一个楼栋，或当前角色不能处理该审批阶段。联系超级管理员核对账号范围。

### 收不到通知

先查看站内通知。微信消息还取决于用户授权、模板配置和外部平台状态；站内通知是系统内的主要结果依据。

### 动态签到码无法使用

确认预约已通过、当前处于允许签到时间、凭证未过期且未使用。如仍失败，由现场管理员核对预约并按权限人工处理。

## 5. 使用规范

- 不借用他人账号或签到凭证；
- 不通过重复提交占用多个座位；
- 不能按时使用时尽早取消；
- 按房间规则控制人数、噪声和设备使用；
- 发现数据或功能异常时提供页面、时间、预约编号和操作步骤，避免提交隐私信息或访问令牌。
