# 轻拟物 3D 智慧空间 UI 重构方案

## 1. 设计定位

本轮重构将敬一书院功能房预约系统升级为“轻拟物 3D 智慧空间风”。设计参考智能家居控制面板的安静、柔和、留白和面板式交互，但不照搬家电遥控器的低信息密度。

最终目标：

- 保留校园智慧空间属性；
- 使用浅雾灰、冷白、科技蓝和冰青；
- 卡片、按钮和图标具备中轻量 3D 体积感；
- 信息层级清晰，保证预约系统的效率；
- 动效克制、流畅，不使用重型游戏化 3D；
- 学生端、管理员小程序和 Vue 管理后台共享同一设计语言。

## 2. 不可违反的界面规则

1. 正式界面禁止显示服务器地址、测试账号、模板 ID、调试开关、开发者说明和内部修正备注。
2. 禁止将“这里仅保留……”“建议修改……”“本页面用于……”等设计或开发说明渲染给用户。
3. 禁止使用 Emoji 或单个汉字替代正式功能图标。
4. 书法或手写气质只允许用于极少数品牌标题点缀；正文、表单、数据和按钮统一使用现代系统字体。
5. 所有浅灰文本必须满足可读性，不得为了“高级感”降低信息对比度。
6. 页面动画必须支持低性能设备，避免同时运行大量滤镜、粒子和无限动画。
7. 不修改已经完成的认证、预约一致性、动态签到、Outbox、审计、备份和发布业务逻辑。

## 3. 字体与排版

统一字体栈：

```css
-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif
```

排版原则：

- 页面主标题：40～48rpx；
- 模块标题：32～36rpx；
- 正文：26～30rpx；
- 辅助信息：20～24rpx；
- 重要数字采用更高字重，但不使用过度压缩字体；
- 正文行高不低于 1.45。

## 4. 颜色体系

- 主科技蓝：`#2F80FF`
- 深科技蓝：`#1767E8`
- 冰青强调：`#4FD5E8`
- 成功：`#39BF78`
- 警告：`#F8A63C`
- 错误：`#FF5F68`
- 页面背景：`#F1F3F7`
- 一级卡片：`#FBFCFE`
- 主文字：`#1F2A3D`
- 次级文字：`#61708A`
- 辅助文字：`#9AA6B8`

## 5. 立体表现

### 卡片

- 24～44rpx 圆角；
- 双向柔和阴影；
- 顶部白色高光描边；
- 轻微冷白渐变；
- 点击时下沉 2～4rpx，并切换为内阴影。

### 图标

统一采用轻量 3D 科技图标：

- 左上光源；
- 柔和圆角底座；
- 低饱和主色；
- 统一透视、阴影方向、尺寸和基线；
- 优先使用本地 SVG 母版，并提供 PNG/WebP 降级资源；
- 不依赖远程图标 URL。

### Hero 区

首页、登录和房间详情可使用 CSS 3D 或本地插画构建空间场景：

- 房间、书桌、门窗或校园建筑微场景；
- 状态浮层；
- 柔光和平台阴影；
- 不使用虚假的学校建筑名称或不真实校门。

## 6. 动效规范

允许：

- 页面进入淡入与上浮；
- 卡片按压和回弹；
- Tab 胶囊滑块；
- 数字轻量滚动；
- 状态点呼吸；
- 成功勾选与少量光点；
- Hero 场景缓慢浮动。

限制：

- 同屏持续动画不超过 3 组；
- 单次过渡建议 160～420ms；
- 无限动画周期不少于 2.4 秒；
- 禁止大范围旋转、频闪、爆炸粒子或重度视差。

## 7. 页面重构顺序

### 第一批：设计系统与核心入口

- 全局 Design Tokens；
- 登录页；
- 学生首页；
- 自定义底部导航；
- 通用卡片、按钮、状态和动效。

### 第二批：预约主流程

- 房间列表；
- 房间详情；
- 自习室座位与时间轴；
- 预约确认；
- 预约详情；
- 我的预约。

### 第三批：学生服务

- 消息；
- 个人中心；
- 信用详情；
- 规则制度；
- 反馈；
- 订阅设置；
- 动态签到二维码。

### 第四批：管理员小程序

- 管理首页；
- 审批；
- 房间管理；
- 用户与信用；
- 反馈；
- 统计；
- 公告；
- 管理员个人中心。

### 第五批：Vue 管理后台

- 登录；
- 布局与导航；
- 工作台；
- 审批、房间、用户、信用和统计；
- 实时监控与运维状态。
