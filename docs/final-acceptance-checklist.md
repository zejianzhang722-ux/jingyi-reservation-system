# 最终验收清单

## 1. 验收信息

验收时记录：

- 发布版本与 Git 提交号；
- API 和管理后台镜像摘要；
- 验收日期、环境和负责人；
- MySQL、Redis、服务器规格和测试数据规模；
- 所有 GitHub Actions 工作流链接；
- 性能、备份、恢复和发布演练证据。

## 2. 认证与权限

- [ ] 学生、管理员、辅导员和超级管理员登录正常；
- [ ] 访问令牌与刷新令牌不能混用；
- [ ] 退出后令牌失效；
- [ ] 普通管理员不能访问其他楼栋数据；
- [ ] 学生不能查看或修改他人预约；
- [ ] 超级管理员专属接口有后端权限校验；
- [ ] 密码、令牌和密钥未出现在日志、前端或仓库中。

## 3. 预约闭环

- [ ] 房间列表、详情、座位和时间轴正常；
- [ ] 普通房间和座位房间均可成功预约；
- [ ] 同一用户重复提交只产生一条预约；
- [ ] 并发请求不能重复占用相同时间槽；
- [ ] 修改、取消、重新预约正常；
- [ ] 候补加入、退出和转正式预约正常；
- [ ] 审批通过和拒绝使用条件更新，重复处理返回冲突；
- [ ] 取消和拒绝后槽位被释放；
- [ ] 学生预约列表和详情字段完整。

## 4. 签到与信用

- [ ] 动态签到凭证具有短有效期；
- [ ] 已使用凭证不能重放；
- [ ] 学生不能调用工作人员签到接口；
- [ ] 扫码、人工签到、签退和状态查询正常；
- [ ] 爽约、违规和良好行为的信用分变动正确；
- [ ] 信用限制和解除规则正常。

## 5. 通知与实时事件

- [ ] 站内通知写入成功；
- [ ] 同一业务事件不会重复生成通知；
- [ ] Outbox 领取、重试和死信正常；
- [ ] 多实例下不会重复投递同一事件；
- [ ] WebSocket 只允许有效身份连接；
- [ ] 房间状态和预约事件可跨实例广播；
- [ ] 外部消息失败不影响核心预约事务提交。

## 6. 管理端

- [ ] 网页后台和小程序管理员端均可登录；
- [ ] 预约审批、房间、座位、公告、反馈和信用管理正常；
- [ ] 管理员列表和统计均遵守楼栋范围；
- [ ] 大列表具有分页和筛选；
- [ ] 导出结果与筛选条件一致；
- [ ] 页面没有测试文案、调试地址或内部说明；
- [ ] 移动端无明显重叠、遮挡和不可点击区域。

## 7. 数据一致性与迁移

- [ ] 预约一致性迁移可重复执行；
- [ ] 通知 Outbox 迁移可重复执行；
- [ ] 审计迁移可重复执行；
- [ ] 备份恢复迁移可重复执行；
- [ ] 性能索引迁移可重复执行；
- [ ] 生产 Readiness 检查没有缺失对象；
- [ ] 迁移前后关键记录数量一致；
- [ ] 发布窗口内新旧应用均兼容扩展后的数据库结构。

## 8. 安全

- [ ] CORS 只允许正式来源；
- [ ] 生产环境必须使用 HTTPS；
- [ ] 上传类型、大小和文件名经过校验；
- [ ] 上传文件通过受控路由读取；
- [ ] 登录、预约和签到具有合理限流；
- [ ] 容器以非 root 用户运行；
- [ ] 应用容器使用只读根文件系统、删除 capabilities 并禁止提权；
- [ ] 数据库和 Redis 不直接暴露到公网；
- [ ] CI 的秘密扫描和安全回归全部通过。

## 9. 可观测性与审计

- [ ] 每个请求具有请求编号；
- [ ] 日志为结构化格式且不含敏感数据；
- [ ] Live、Ready、Version、Status 和 Metrics 接口正常；
- [ ] HTTP 延迟、错误率、内存、Outbox、备份和数据库指标可采集；
- [ ] 数据库慢查询只记录脱敏指纹；
- [ ] 连接池排队、慢查询比例和数据库错误率告警正常；
- [ ] 管理和敏感操作写入审计日志；
- [ ] 审计哈希链校验通过；
- [ ] 修改审计记录能够被完整性检查发现。

## 10. 备份与灾备

- [ ] 备份包含 MySQL 数据和上传文件；
- [ ] 备份使用认证加密并具有外部和组件级校验；
- [ ] 第二副本位于不同故障域；
- [ ] 自动备份和保留策略正常；
- [ ] 最近成功备份未超过告警时限；
- [ ] 隔离恢复能够通过必要数据表和数量检查；
- [ ] 灾备演练记录实际 RPO 和 RTO；
- [ ] 默认不能直接覆盖生产数据库；
- [ ] 数据归档在删除源数据前已生成可验证文件。

## 11. 发布与回滚

- [ ] API、Scheduler 和管理后台镜像可复现构建；
- [ ] `/ops/version` 返回预期提交号和发布编号；
- [ ] 发布前自动创建并验证备份；
- [ ] 数据库迁移和 Readiness 门禁通过；
- [ ] 新版本先部署到非活动槽位；
- [ ] 流量切换前新槽位 Ready 且版本身份正确；
- [ ] Nginx 配置检查后无中断切换；
- [ ] 上一个槽位在观察窗口内保留；
- [ ] 应用回滚演练成功；
- [ ] 回滚不执行未经验证的反向数据库迁移。

## 12. 性能与容量

- [ ] 性能遥测回归通过；
- [ ] 所有必要复合索引存在；
- [ ] 代表性查询执行计划可使用预期索引；
- [ ] 数据库查询 P95 满足基线；
- [ ] HTTP 容量烟雾测试满足基线；
- [ ] 预约并发一致性测试通过；
- [ ] 峰值测试期间没有持续连接池排队；
- [ ] Outbox、Redis 和 MySQL 在降载后及时恢复；
- [ ] 生产规格峰值测试保留至少 30% 余量；
- [ ] 测试报告、服务器规格、数据规模和提交号已归档。

## 13. 文档

- [ ] README 与实际启动方式一致；
- [ ] API 接口手册已更新；
- [ ] 数据字典已更新；
- [ ] 学生和管理员用户手册已更新；
- [ ] 容器发布与回滚手册已更新；
- [ ] 备份恢复与灾备手册已更新；
- [ ] 性能容量手册已更新；
- [ ] 环境变量模板没有真实密钥；
- [ ] 所有脚本命令可在干净环境执行。

## 14. 正式发布判定

仅在以下条件全部满足时标记正式验收通过：

1. 所有必需 CI 均为成功；
2. 没有未处理的高危或阻断问题；
3. 备份恢复和应用回滚均完成演练；
4. 性能门槛满足且保留容量余量；
5. 生产配置由两人复核；
6. 负责人签署验收记录；
7. 发布后观察窗口内没有新增 Critical 告警。
