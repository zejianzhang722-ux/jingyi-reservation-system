# 工程化改造路线图

本路线图将预约系统的生产化改造划分为十个阶段。阶段编号用于控制改造边界、测试范围和合并顺序，不代表产品功能停止演进。

| 阶段 | 主题 | 状态 |
|---|---|---|
| 第一阶段 | 认证、授权、资源归属与HTTP语义修复 | 已完成 |
| 第二阶段 | 动态签到二维码与一次性凭证防重放 | 已完成 |
| 第三阶段 | 生产数据库隔离、预约并发一致性与幂等事务 | 已完成 |
| 第四阶段 | 多实例运行协调、Redis分布式锁与实时事件治理 | 已完成 |
| 第五阶段 | 管理员数据范围、上传与静态资源安全、秘密治理 | 已完成 |
| 第六阶段 | 通知幂等、事务Outbox、重试、死信与跨实例投递 | 已完成 |
| 第七阶段 | 请求关联、结构化日志、运行指标、审计哈希链与告警 | 已完成 |
| 第八阶段 | 自动备份、恢复校验、灾备演练、数据保留与归档 | 已完成 |
| 第九阶段 | 容器化部署、灰度发布、数据库发布门禁和一键回滚 | 已完成 |
| 第十阶段 | 性能压测、容量基线、体验收尾、文档和最终验收 | 已完成 |

**十个阶段已全部完成。** 后续工作进入常规版本迭代、生产部署验证和持续容量复测。

## 第十阶段完成内容

- 增加数据库查询次数、错误率、慢查询和延迟直方图；
- 增加数据库连接池总连接、空闲连接和等待队列监控；
- 增加数据库错误率、慢查询比例和连接池排队告警；
- 慢查询日志采用脱敏 SQL 指纹，不记录查询参数；
- 预约详情合并座位查询，消除一次额外数据库访问；
- 新增房间、座位、预约、签到、通知和候补复合索引；
- 新增可重复性能索引迁移、启动门禁和生产 Readiness 校验；
- 新增真实 MySQL 执行计划、索引和数据库延迟回归；
- 新增启动后 API 的有界 HTTP 性能烟雾测试；
- 新增独立性能与容量 GitHub Actions 工作流并保留测试证据；
- 完成预约高峰、签到、审批、通知和报表负载模型；
- 完成性能容量、API、数据字典、用户使用和最终验收文档；
- 将性能迁移加入蓝绿发布数据库门禁；
- 正式发布版本保持为 `1.0.0`。

## 持续维护要求

- 服务器规格、数据规模或部署拓扑变化后重新建立容量基线；
- 破坏性数据库变更继续采用“扩展—迁移—收缩”；
- 每次发布必须通过全部必需 CI、发布前备份和 Readiness 门禁；
- 每季度执行恢复演练和应用回滚演练；
- 生产峰值应保留至少 30% 容量余量；
- 新功能必须同步更新接口、数据字典、用户手册和验收清单。
