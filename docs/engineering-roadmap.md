# 工程化改造路线图

本路线图将当前预约系统的生产化改造收敛为十个阶段。阶段编号用于控制改造边界、测试范围和合并顺序，不代表产品功能停止演进。

| 阶段 | 主题 | 状态 |
|---|---|---|
| 第一阶段 | 认证、授权、资源归属与HTTP语义修复 | 已完成 |
| 第二阶段 | 动态签到二维码与一次性凭证防重放 | 已完成 |
| 第三阶段 | 生产数据库隔离、预约并发一致性与幂等事务 | 已完成 |
| 第四阶段 | 多实例运行协调、Redis分布式锁与实时事件治理 | 已完成 |
| 第五阶段 | 管理员数据范围、上传与静态资源安全、秘密治理 | 已完成 |
| 第六阶段 | 通知幂等、事务Outbox、重试、死信与跨实例投递 | 已完成 |
| 第七阶段 | 请求关联、结构化日志、运行指标、审计哈希链与告警 | 已完成 |
| 第八阶段 | 自动备份、恢复校验、灾备演练、数据保留与归档 | 已完成 |
| 第九阶段 | 容器化部署、灰度发布、数据库发布门禁和一键回滚 | 本次完成 |
| 第十阶段 | 性能压测、容量基线、体验收尾、文档和最终验收 | 待完成 |

第九阶段完成后，按本路线图还剩 **1个阶段**。

## 第九阶段完成内容

- API与Scheduler共用的可复现生产镜像；
- 固定数值非root用户、只读根文件系统、能力删除和禁止提权；
- 管理后台多阶段构建与非特权Nginx镜像；
- MySQL、Redis、双API槽位、Scheduler、Admin和Gateway生产拓扑；
- 公共版本端点，返回版本、Git提交、构建时间、发布编号和部署槽位；
- 发布前真实加密备份及完整性校验；
- 四组数据库迁移与Readiness发布门禁；
- 非活动槽位部署、健康等待、构建身份核对和Nginx无中断切换；
- 上一槽位保留与应用级快速回滚；
- 破坏性数据库变更的扩展—迁移—收缩约束；
- 容器构建、运行用户、健康检查、Compose拓扑和发布脚本CI校验；
- 完整容器发布与回滚手册。

## 第十阶段验收目标

- 预约高峰、签到、审批、通知和报表的压测模型；
- 延迟、吞吐、连接池、Redis与数据库容量基线；
- 慢查询、热点接口和大列表优化；
- 学生端、管理员小程序和管理后台的最终体验回归；
- 部署、运维、数据字典、接口和用户手册齐备；
- 完整验收清单和正式版本发布。
