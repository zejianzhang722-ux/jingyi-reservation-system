# 工程化改造路线图

本路线图将当前预约系统的生产化改造收敛为十个阶段。阶段编号用于控制改造边界、测试范围和合并顺序，不代表产品功能停止演进。

| 阶段 | 主题 | 状态 |
|---|---|---|
| 第一阶段 | 认证、授权、资源归属与HTTP语义修复 | 已完成 |
| 第二阶段 | 动态签到二维码与一次性凭证防重放 | 已完成 |
| 第三阶段 | 生产数据库隔离、预约并发一致性与幂等事务 | 已完成 |
| 第四阶段 | 多实例运行协调、Redis分布式锁与实时事件治理 | 已完成 |
| 第五阶段 | 管理员数据范围、上传与静态资源安全、秘密治理 | 已完成 |
| 第六阶段 | 通知幂等、事务Outbox、重试、死信与跨实例投递 | 已完成 |
| 第七阶段 | 请求关联、结构化日志、运行指标、审计哈希链与告警 | 本次完成 |
| 第八阶段 | 自动备份、恢复校验、灾备演练、数据保留与归档 | 待完成 |
| 第九阶段 | 容器化部署、灰度发布、数据库发布门禁和一键回滚 | 待完成 |
| 第十阶段 | 性能压测、容量基线、体验收尾、文档和最终验收 | 待完成 |

第七阶段完成后，按本路线图还剩 **3个阶段**。

## 第八阶段验收目标

- MySQL全量与增量备份脚本；
- 上传文件与配置的配套备份；
- 备份加密、校验和、保留周期和异地副本；
- 在隔离数据库中自动恢复并执行一致性检查；
- 明确RPO、RTO及季度灾备演练流程；
- 替换当前仅统计数量的“备份”接口，避免形成虚假的备份成功状态。

## 第九阶段验收目标

- 可复现的容器镜像和最小权限运行用户；
- API、Scheduler Worker、MySQL、Redis的部署拓扑；
- 迁移前置检查、逐步放量和自动回滚门禁；
- 蓝绿或金丝雀发布、兼容性窗口和回滚手册；
- 版本、提交号和构建信息可查询。

## 第十阶段验收目标

- 预约高峰、签到、审批、通知和报表的压测模型；
- 延迟、吞吐、连接池、Redis与数据库容量基线；
- 慢查询、热点接口和大列表优化；
- 学生端、管理员小程序和管理后台的最终体验回归；
- 部署、运维、数据字典、接口和用户手册齐备；
- 完整验收清单和正式版本发布。
