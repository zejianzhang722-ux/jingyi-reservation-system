# 数据字典

## 基本约定

数据库采用 MySQL 8.0 和 `utf8mb4`。业务时间由服务端生成；结构变更必须通过可重复迁移脚本执行。JSON 字段仅用于扩展元数据，需要筛选、排序或约束的数据应使用正式字段。

## 核心资源

| 数据表 | 用途 | 主要字段 |
|---|---|---|
| `buildings` | 楼栋基础数据 | 名称、地址、楼层数 |
| `rooms` | 功能房配置 | 类型、楼栋、位置、容量、开放时间、审批要求、状态 |
| `seats` | 座位配置 | 房间、座位号、行列、状态、电源 |
| `system_config` | 系统参数 | 唯一配置键、配置值、说明 |
| `announcements` | 公告 | 标题、正文、类型、置顶、发布状态 |

## 用户与权限

| 数据表 | 用途 | 主要字段 |
|---|---|---|
| `users` | 学生和业务用户 | OpenID、学号、姓名、楼栋、角色、信用分、限制状态 |
| `admins` | 管理员账号 | 用户名、密码摘要、角色、管理楼栋、状态 |
| `credits_log` | 信用分流水 | 用户、分数变动、变动后分数、类型、关联记录 |
| `violations` | 违规记录 | 用户、类型、说明、扣分、创建人 |

角色包括学生、普通管理员、辅导员和超级管理员。普通管理员的数据范围由楼栋字段控制。

## 预约与候补

| 数据表 | 用途 | 主要字段与约束 |
|---|---|---|
| `reservations` | 预约主记录 | 用户、房间、座位、日期、起止时间、状态、预约码、幂等键、审批信息 |
| `reservation_slots` | 预约离散时间槽 | 房间、座位范围、日期和分钟槽组合唯一 |
| `reservation_waitlist` | 候补队列 | 用户、房间、座位、日期、时间、候补状态 |
| `reservation_groups` | 团体预约 | 房间、日期、时间、目的、创建人、状态 |
| `reservation_group_members` | 团体成员 | 团体、用户、座位、确认状态 |

预约常用状态：待审批、辅导员待审批、已通过、已签到、已完成、已拒绝、已取消和爽约。

核心唯一约束：

- 用户与幂等键唯一，避免重复创建预约；
- 房间、座位范围、日期和分钟槽唯一，避免并发重复占用；
- 等待状态下同一用户和同一时段的候补记录唯一。

## 签到与空间使用

| 数据表 | 用途 | 主要字段 |
|---|---|---|
| `checkins` | 签到签退事实 | 预约、用户、房间、签到时间、签退时间、签到类型 |
| `checkin_credentials` | 动态签到凭证 | 预约、随机标识、过期时间、使用时间 |
| `reading_room_logs` | 阅览室进出 | 用户、进入时间、离开时间 |

## 通知

| 数据表 | 用途 | 主要字段 |
|---|---|---|
| `notifications` | 站内通知 | 用户、类型、标题、正文、业务数据、去重键、已读状态 |
| `notification_outbox` | 异步投递队列 | 事件键、渠道、负载、状态、重试次数、锁定信息、错误信息 |

Outbox 状态包括待发送、处理中、已发送、失败和死信。事件键与通知去重键用于避免重复投递。

## 内容与运营

| 数据表 | 用途 | 主要字段 |
|---|---|---|
| `posters` | 海报申请 | 申请人、组织、时间、位置、图片、审核状态 |
| `feedbacks` | 用户反馈 | 类型、内容、联系方式、状态、回复、处理人 |
| `rules` | 使用规则 | 类型、标题、正文、启用状态、排序 |

## 审计、备份与归档

| 数据表 | 用途 | 主要字段 |
|---|---|---|
| `operation_logs` | 操作审计 | 请求编号、操作者、动作、目标、结果、元数据、前序哈希、当前哈希 |
| `backup_runs` | 备份执行记录 | 备份编号、触发方式、状态、文件、大小、校验值、第二副本、错误信息 |
| `data_archives` | 归档登记 | 归档编号、源表、截止时间、行数、文件、校验值、状态 |

审计日志通过哈希链发现历史记录被修改的情况。归档只有在加密文件及校验文件成功写入后，才允许清理源数据。

## 主要性能索引

- 房间：状态、类型和楼栋；
- 座位：房间、状态和展示顺序；
- 预约：审批状态与创建时间；
- 预约：用户、状态和日程；
- 预约：房间、日期、状态、座位和时间；
- 签到：房间和签到时间；
- 通知：用户、已读状态和创建时间；
- 候补：状态和创建时间。

## 数据关系

- 一个楼栋包含多个房间；
- 一个房间包含多个座位和预约；
- 一个用户可拥有多个预约、通知、签到和信用流水；
- 一个预约对应多个离散时间槽；
- 一个站内通知可产生多个渠道投递事件；
- 审计、备份和归档记录独立于普通业务数据生命周期保存。

## 变更规则

1. 发布使用“扩展—迁移—收缩”流程，旧应用仍在回滚窗口内时不得删除旧字段；
2. 添加唯一约束前先检查重复数据；
3. 添加索引前评估锁表时间和存储空间；
4. 恢复默认写入隔离数据库；
5. 测试、生产和备份使用相互隔离的数据库与目录。
