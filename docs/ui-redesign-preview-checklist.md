# 轻拟物 3D 智慧空间视觉重构预览检查清单

本文档用于 PR #14 在微信开发者工具、真机预览和后续合并前的 UI 验证。当前分支以 WXSS 视觉迁移为主，原则上不改动预约、审批、登录、统计、反馈等业务逻辑。

## 1. 编译与基础检查

- [ ] 微信开发者工具可正常编译，无 WXSS / WXML / JS 报错。
- [ ] 小程序启动页、登录页、学生端首页、管理员端首页均可进入。
- [ ] 控制台无由样式选择器、图片路径、组件属性引起的明显错误。
- [ ] 页面中不出现测试账号、服务器地址、内部修正说明、AI 中间提示词或非用户展示文案。

## 2. 全局视觉一致性

- [ ] 所有主要页面背景统一为浅色渐变或浅色拟物背景。
- [ ] 卡片统一使用白色/浅蓝拟物面板、柔和阴影、圆角和轻微高光。
- [ ] 主按钮统一为蓝青渐变胶囊按钮，危险操作使用浅红胶囊或红色标签。
- [ ] 状态标签统一为胶囊样式，待审核/已通过/已完成/爽约/封禁等状态色清晰区分。
- [ ] 不使用 Emoji 或单字替代正式图标；图标应以 CSS 图形、渐变底座或正式图标资源呈现。

## 3. 学生端主流程检查

- [ ] 登录页：学生/管理员模式切换正常，按钮不被安全区遮挡。
- [ ] 首页：Hero、分类卡、统计卡、公告卡在小屏和长文本下不重叠。
- [ ] 房间列表/详情：长房间名、长位置、容量、设备标签换行正常。
- [ ] 预约确认：输入框、协议勾选、底部确认按钮在 iPhone 刘海屏和低高度屏幕上可点击。
- [ ] 我的预约：各状态卡片、取消/签到按钮、空状态、加载态正常。
- [ ] 预约详情：状态、时间、用途、凭证码入口和操作按钮排版正常。
- [ ] 动态二维码：二维码/凭证占位、失效遮罩、刷新按钮正常显示。
- [ ] 个人中心：头像、信用分、菜单列表、退出登录正常显示。
- [ ] 个人信息编辑：禁用字段、可编辑字段、保存按钮样式统一。
- [ ] 信用明细/使用统计：信用仪表、记录卡、统计卡、空状态正常。

## 4. 学生端扩展页面检查

- [ ] 消息页：分类筛选、未读标记、消息卡片、长消息摘要正常。
- [ ] 制度页：长制度内容可阅读，内凹文本区无横向溢出。
- [ ] 帮助反馈：FAQ 展开/收起、反馈表单和提交按钮正常。
- [ ] 网络设置：局域网地址、端口、完整接口地址输入区域不溢出。
- [ ] 订阅设置：授权状态、模板列表和授权按钮正常。
- [ ] 团体预约：成员列表、成员标签、分享按钮和团体信息卡正常。
- [ ] 海报申请：图片上传区域、预览图、表单项和提交按钮正常。
- [ ] 空间对比：选中房间胶囊、对比表格、底部选择面板在小屏可用。

## 5. 管理员端检查

- [ ] 管理员首页：数据入口、菜单入口、管理员身份信息正常显示。
- [ ] 管理中心：各管理入口卡片视觉统一，点击跳转正常。
- [ ] 预约审批：搜索、状态筛选、通过/拒绝按钮和长申请信息正常。
- [ ] 宿生管理：搜索、信用分、封禁/解封、调分按钮正常。
- [ ] 信用管理：违规记录、黑名单、配置摘要三类 Tab 正常切换。
- [ ] 功能房管理：分类筛选、房间状态、开放/关闭按钮正常。
- [ ] 反馈管理：回复表单、已处理状态、标记处理按钮正常。
- [ ] 公告管理：发布面板、类型选择、公告内容、删除按钮正常。
- [ ] 数据统计：统计卡、排行列表、爽约概览、信用分布正常。
- [ ] 管理员个人中心：管理员信息、菜单项、退出登录正常。

## 6. 适配与交互检查

- [ ] iPhone 小屏、标准屏、大屏预览均无横向滚动。
- [ ] 底部固定按钮与自定义 TabBar 不重叠。
- [ ] 页面底部安全区 `env(safe-area-inset-bottom)` 生效。
- [ ] 长姓名、长学号、长房间名、长公告标题不会挤压按钮。
- [ ] 卡片点击、按钮点击、Tab 切换、底部弹层出现/关闭动效自然。
- [ ] 阴影、渐变和 blur 在微信开发者工具及真机上性能可接受。

## 7. 头像问题单独复测

当前头像持久化暂标记为“需正式持久化存储环境复测”。请单独确认：

- [ ] 正式 MySQL 环境下，`users.avatar` 字段能保存上传后的 `/uploads/...` 或对象存储地址。
- [ ] 上传目录 `server/uploads/` 在后端重启、重新部署后不会被清空。
- [ ] 如使用云服务器，应将头像文件迁移到持久化磁盘、静态资源目录或对象存储。
- [ ] 重新登录后 `/user/profile` 返回的 `avatar` 字段不是空字符串。
- [ ] 小程序端 `auth.normalizeAssetUrl` 拼接后的头像地址可访问且不 404。
